img src = "이미지경로" alt = "이미지에 대한 설명"

1. img 요소는 스스로 닫는 형태의 인라인 요소
2. alt 속성은 이미지를 대체하는 텍스트, 이미지 내용을 텍스트로 넣는다.
3. alt 속성에 입력된 대체 텍스트는 검색 엔진, 스크린리더기 등에서 활용한다.
4. alt 속성은 반드시 선언 되어야 하며, 선언되지 않을 경우 비표준이다.
5. 이미지가 아무런 내용을 담지 않는 경우 (예: 블릿)에도 alt 속성은 선언하돼 빈값 또는 #표시
6. width, height는 이밎의 기본 크기를 명시하는 속성, 요즘은 css로 대체하는 경우가 많다.

응용 소프트웨어가 운영될 환경과 유사한 구조로 구축
분석 단계의 산출물을 바탕으로 하드웨어, 소프트웨어를 선정
하드웨어 환경: 사용자와의 인터페이스 역할을 하는 client, client와 통신하여 서비스를 제공하는
 server로 구성된다.

파일 서버 - 데이터베이스에 저장하기에 비효율적이거나, 서비스 제공을 목적으로 유지하는 파일들을 저장함.

개발 소프트웨어의 종류
요구사항관리도구 - 요구사항의 수집, 분석, 추적 등을 편리하게 도와주는 소프트웨어
설계/모델링도구 - UML(통합 모델링 언어, Unified Modeling Language)을 지원하며, 개발의
		 전 과정에서 설계 및 모델링을 도와주는 소프트웨어
구현도구 - 개발 언어를 통해 애플리케이션의 실제 구현을 지원하는 소프트웨어
빌드도구 - 구현 도구를 통해 작성된 소스의 빌드 및 배포, 라이브러리 관리를 지원하는 소프트웨어
테스트도구 - 모듈들이 요구사항에 적합하게 구현되었는지 테스트하는 소프트웨어
형상 관리도구 - 산출물들을 버전별로 관리하여 품질 향상을 지원하는 소프트웨어 ex) github


웹서버의 기능
HTTP / HTTPS - 브라우저로부터 요청을 받아(request) 응답(response)할 때 사용되는 프로토콜ㄴ
통신기록(Communication Log) - 처리한 요청들을 로그 파일로 기록하는 기능
정적 파일 관리(Managing Static Files) - HTML, CSS, 이미지 등에 정적 파일들을 저장하고 관리하는 기능
대역폭의 제한(BandWidth Throttling) - 네트워크 트래픽의 포화를 방지하기 위하여 응답 속도를 제한하는 기능
가상 호스팅(Virtual Hosting) - 하나의 서버로 여러 개의 도메인 이름을 연결하는 기능
인증(Authentication) - 사용자가 합법적인 사용자인지를 확인하는 기능

개발언어 선정기준
적정성 - 소프트웨어의 목적에 적합해야 함
효율성 - 코드의 작성 및 구현이 효율적이어야 한다.
이식성 - 다양한 시스템 및 환경에 적용이 가능해야 함
친밀성 - 개발 언어에 대한 개발자들의 이해도와 활용도가 높아야 함
범용성 - 다른 개발 사례가 존재하고 여러분야에서 활용되고 있어야 함.
